# Manhwa SFX Cleaner & Translator 🖌️💬

Автоматизована система для очищення манхви від звукових ефектів (SFX) та їхнього перекладу на українську мову з використанням комп'ютерного зору та генеративного ШІ.

## 🚀 Основні можливості

* **Сегментація та очищення**: Виявлення звукових ефектів за допомогою YOLO та інтелектуальне видалення їх через Inpainting (модель MAT).
* **VLM Інтерпретація**: Опис візуальної сцени за допомогою моделі Qwen2.5-VL для кращого розуміння контексту звуку.
* **RAG Система**: Використання векторної бази даних ChromaDB для пошуку найбільш відповідних українських ономатопей.
* **Стилізований рендеринг**: Генерація тексту за допомогою Stable Diffusion 1.5 + ControlNet + IP-Adapter для збереження оригінального стилю та текстури корейських ієрогліфів.
* **Двоетапний конвеєр**: Поділ процесу на етап очищення (VRAM-оптимізовано) та етап генерації/перекладу.

## 🏗️ Архітектура проєкту

Система складається з декількох ключових модулів:

### 1. Етап очищення (Cleanup Stage)
* **`segmentation.py`**: Використовує YOLO для детекції масок SFX. Включає покращення масок за допомогою алгоритму GrabCut та розмиття для м'яких країв.
* **`inpainting.py`**: Реалізує модель MAT (Mask-Aware Transformer) для заповнення порожнеч після видалення тексту.
* **`clean_pipeline.py`**: Координує роботу сегментації та інпейнтингу, керуючи очищенням пам'яті GPU.

### 2. Етап перекладу та рендерингу (VLM & Rendering)
* **`vlm_interpreter.py`**: Логіка перекладу. Використовує Qwen2.5-VL для аналізу зображення та семантичний пошук через `SentenceTransformer` для підбору українського відповідника.
* **`st_render.py`**: Найскладніший модуль. Витягує геометрію оригінального тексту (кут нахилу, розмір), створює маску українського тексту та накладає текстуру за допомогою Stable Diffusion, щоб новий текст виглядав як частина малюнка.

### 3. Оркестрація
* **`full_pipeline.py`**: Головний скрипт, який запускає підпроцеси у різних віртуальних середовищах (якщо потрібно розділити залежності).
* **`sub_cleanup.py`** та **`sub_render.py`**: Точки входу для окремих етапів.

## 🛠️ Технологічний стек

* **Core**: Python 3.10+
* **ML Frameworks**: PyTorch, Ultralytics (YOLOv11/v8)
* **Generative AI**: Diffusers (SD 1.5, ControlNet, IP-Adapter), Transformers (Qwen2.5-VL)
* **Computer Vision**: OpenCV, Pillow
* **Database**: ChromaDB (для RAG)
* **Оптимізація**: BitsAndBytes (4-bit квантизація), `torch.cuda.empty_cache()`

## 📦 Встановлення та запуск

1.  **Налаштування середовища**:
    Рекомендується створити два окремих venv (наприклад, `venv_cleanup` та `venv_render`) через специфічні вимоги `iopaint` та `diffusers`.

2.  **Запуск через головний пайплайн**:
    ```bash
    python full_pipeline.py
    ```

## 📝 Важливі примітки
* **Fix Hub**: У проєкті використовується `fix_hub.py` для забезпечення сумісності старіших версій `diffusers` з оновленим API Hugging Face.
* **Пам'ять**: Проєкт використовує "Lazy Loading" (ліниве завантаження) моделей. Моделі завантажуються у VRAM лише перед безпосереднім використанням і вивантажуються після завершення кроку.
